This is the nurse connect template set

For the purposes of this exercise we will be using a very bare bones gulp based stack.

Build tools

  • Gulp is used to run the build
  • By running gulp or gulp browser-sync you will start up a light wait browsersync static server that will livereload (auto refresh) as new content is added and inject new styling everytime the styling documents are saved.
  • We are using sass as a css preprocessor, with the following conventions:
    • BEM spicifically the SUITcss implementation
    • component based scss partials should be writen to improve reuse

Folder structure

the folder structure follows the following patterns:

  • dist and src directories seporate compiled from source files
  • inside src the sass folder contains a rather standard sass folder structure in line with thesassway.com/how-to-structure-a-sass-project
  • a templates folder is used to contain all html temlplates

Contributing:

please adhere the the above guidlines when contributing. Furthermore dont hesitate to ask other team members or the team lead if anything is unclear. the simpilest way to contribute is as follows:

  1. duplicate /templates/base.html and rename to a temp component your working on or a new page.
  2. create a new file in /src/sass/partials/_<conponent-name>.scss if necessary
  3. add an @import for your new partials to the list of partials (alphebetically) in /src/sass/styles.scss.
